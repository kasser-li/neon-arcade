# Task Progress Skill

## 用途
规范长任务的执行流程，确保及时汇报进度，避免消息堆积和静默执行。

## 规则

### 1. 任务分级
- **短任务** (< 1分钟)：直接执行，完成后汇报
- **中任务** (1-3分钟)：执行前说"开始执行XXX"
- **长任务** (> 3分钟)：必须分阶段汇报

### 2. 长任务流程
```
开始 → 进度(30%) → 进度(60%) → 进度(90%) → 完成
```

### 3. 消息模板

**开始任务**：
```
开始执行 [任务名]
预计时间：[X分钟]
步骤：[1. xxx 2. xxx 3. xxx]
```

**进度汇报**：
```
[任务名] 进度 [X%]
已完成：[xxx]
进行中：[xxx]
预计剩余：[X分钟]
```

**任务完成**：
```
[任务名] 完成 ✅
结果：[xxx]
耗时：[X分钟]
```

### 4. 禁止行为
- ❌ 多条消息粘在一起发
- ❌ 长任务中间不汇报
- ❌ 执行完才说在干嘛
- ❌ 一次性发超过 3 条消息

### 5. 消息节奏控制
- 每两条消息间隔至少 5 秒
- 复杂任务分多条消息，每条一个重点
- 给用户留出回复时间

## 使用示例

用户：重写2048和飞机大战

正确流程：
```
AI: 开始执行游戏重写任务
    预计时间：15分钟
    步骤：1. 重写2048 2. 重写飞机大战 3. 测试验证

AI: [5分钟后]
    游戏重写 进度 30%
    已完成：2048基础逻辑
    进行中：添加触摸控制
    预计剩余：10分钟

AI: [10分钟后]
    游戏重写 进度 70%
    已完成：2048完成，飞机大战基础逻辑
    进行中：飞机大战触摸控制
    预计剩余：5分钟

AI: [15分钟后]
    游戏重写 完成 ✅
    结果：2048和飞机大战都已重写，支持手机触摸
    耗时：15分钟
```

## 自检
每次执行长任务前，问自己：
1. 这是长任务吗？需要分阶段汇报吗？
2. 用户现在知道我在干嘛吗？
3. 消息会不会太多太密？
